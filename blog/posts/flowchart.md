## 什么是流程图？

流程图用于描述一个过程的步骤与决策路径，适合展示用户操作流程、业务审批流程、算法逻辑等场景。Mermaid 使用 `graph` 或 `flowchart` 关键字声明流程图，语法简洁，无需绘图工具。

## 基本语法

### 声明图表方向

```
graph TD
```

方向关键字：`TD`（从上到下）、`LR`（从左到右）、`BT`（从下到上）、`RL`（从右到左）。

### 节点类型

```
graph TD
    A[方框节点]
    B(圆角方框)
    C{菱形判断}
    D((圆形))
    E([椭圆/开始结束])
```

| 写法 | 形状 | 常见用途 |
|------|------|---------|
| `A[文字]` | 方框 | 普通步骤 |
| `A(文字)` | 圆角方框 | 子流程 |
| `A{文字}` | 菱形 | 判断/条件 |
| `A((文字))` | 圆形 | 连接点 |
| `A([文字])` | 椭圆 | 开始/结束 |

### 连线与标签

```
graph TD
    A --> B
    A -->|有标签| C
    A --- D
    A -.-> E
    A ==> F
```

- `-->` 实线箭头（最常用）
- `-->|文字|` 带标签的箭头
- `---` 无箭头实线
- `-.->` 虚线箭头
- `==>` 粗线箭头

## 完整示例：用户登录流程

```
graph TD
    A([开始]) --> B[用户输入账号密码]
    B --> C{账号是否存在?}
    C -->|否| D[显示错误提示]
    D --> B
    C -->|是| E{密码是否正确?}
    E -->|否| F[记录失败次数]
    F --> G{超过3次?}
    G -->|是| H[锁定账号]
    G -->|否| B
    E -->|是| I[登录成功]
    I --> J([结束])
```

## 常用语法速查

| 语法 | 含义 |
|------|------|
| `graph TD` | 从上到下的流程图 |
| `graph LR` | 从左到右的流程图 |
| `A --> B` | 带箭头连线 |
| `A -->|标签| B` | 带文字标签的连线 |
| `A -.-> B` | 虚线箭头 |
| `subgraph 标题` | 子图/分组 |
